      Air Quality Monitor
      Description Air Quality Monitor
      People User:DocOcassi
      Theme Electronics
      Website
      Status Active


      An Air Particulate Monitor based on the design from https://luftdaten.info/en/home-en/ the Aberdeen air quality project has a website https://airaberdeen.org where you should find information and contact about the wider project, this is an arcive of some information related to the sensor kit, not particularly structured

      The sensor kit monitors PM2.5 and PM10 particulates along with humidity and temperature. Based of a infra red scatter and wavelength sensing to determine particulate size.

      Deployment

      We have a number of sensors up in the Aberdeen area, you can see the geographical map of our current registered sensors in the link below.

      • http://maps.luftdaten.info/#13/57.1366/-2.0670

      Construction

      The original Design -> https://luftdaten.info/en/construction-manual/

      Luftdaten believes this provides better air flow for the sensor, under no circumstances do we deny this claim!

      We have a modified design for the sensor which is shown in the picture at the top of the page. This will be the design used in our build workshops.

      A provisional manual for the Build of the sensor can be found https://wiki.57north.org.uk/images/f/f3/LuftdatenTorryBuild.pdf

      Flashing the devices

      If the device needs re-flashed, the apple desktop in the Hackerspace can flash the devices, by logging in as 'north57' and running the shell script in '~/Projects/FineDustSensor'

      Registration

      On the LuftDaten build page at https://luftdaten.info/en/construction-manual/ - there is a form to register the device at the bottom of the page, filling this out will place it on the map http://maps.luftdaten.info/#13/57.1366/-2.0670

      Archived Data

      The data is sent with the long form ID of the sensor to medavi you can find the archived data at:

      • https://www.madavi.de/sensor/csvfiles.php

      A subset of the data is then pulled to the luftdaten archive for display on the map but this is now indexed by the shorter ID there doesn't seem to be a way to know the relation without prior knowledge.

      • http://archive.luftdaten.info/

      Troubleshooting

      Location

      Ensure the device is not located near laundry vents these can affect the readings from the device, also not close to where generators are run. These sort of things can make analysis more complicated.

      Router

      There has been some issues connecting the device to a BT Router, Ed can you get the model to crosscheck
